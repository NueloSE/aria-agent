### Type scale

| Role | Class | Use |
|---|---|---|
| Display | `font-serif text-5xl tracking-tight` | Hero only ‚Äî the ARIA wordmark / headline number |
| H1 (page) | `text-3xl font-semibold tracking-tight` | Page title |
| H2 (section) | `text-xl font-semibold` | Section breaks |
| H3 (subsection) | `text-base font-medium` | Card titles |
| Body | `text-sm` | Default UI text |
| Small / caption | `text-xs text-muted-foreground` | Meta, timestamps |
| Mono | `font-mono tabular-nums` | ALL numbers, addresses, tx hashes, reasoning |

Rule: Instrument Serif appears at most once per screen. Everything that updates in
place (prices, PnL, drawdown) is `font-mono tabular-nums` so digits never jitter.

Utility: `bg-accent-gradient`. **Use for:** the single primary highlight per screen (hero number underline, active-regime badge). **Not for:** body text, buttons that perform destructive actions.

## Tone and voice

### Words to use

Direct, specific, number-forward, factual. Prefer verbs over adjectives, nouns over
metaphors. When describing an action, say what it does: "Closed 2 positions to USDT"
not "Protected your capital." The decision log IS the voice ‚Äî plain reasoning, stated
confidence, no drama.

### Words to avoid

Hype words (revolutionary, game-changing, supercharge), emojis in UI copy,
exclamation marks, overpromising. "Unlock" is banned. Never say "AI-powered" in the
interface ‚Äî show the reasoning instead.

### Voice example

> Regime: high risk ‚Äî F&G 16, market ‚àí21% / 30d. Holding stables. Next check 14:30 UTC.

## Usage dos and don'ts

**Do:**

- Use token utilities (`bg-background`, `text-foreground`, `bg-primary`, `text-gain`) everywhere. Never hardcode hex.
- Keep the 4px spacing grid (`gap-2`, `p-4`, `px-6`).
- `font-mono tabular-nums` for any number that updates in place.
- Test components in both dark (default) and light mode.

**Don't:**

- Hardcode colors (`#FF0000`, `rgb(...)`) in component files.
- Use `transition: all` ‚Äî specify `transition-colors`, `transition-transform`, etc.
- Use more than one gradient per screen, or Instrument Serif more than once per screen.
- Use red/green for anything other than loss/gain semantics.
